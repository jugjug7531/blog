Hexoのブログシェアがうまくいかない

原因：リンク先がおかしい

Hexoのブログ記事の最後にあるShareボタンでTwitter等にシェアした後,
シェアしたリンクをクリックしても記事のページへ飛ぶことができない問題が発生.

シェアしたリンク先のアドレスをよく見ると
「http://jugjug7531.github.io/blog.com/blog.com/2017/01/09/テスト/」
とリポジトリ名が二重になっている.

試行錯誤した挙句, _config.ymlの#URLの部分を

のように書き換えるとうまくいった.

コメントには「…, set url as ‘http://yoursite.com/child’ and …」と書いてあるし, 他のサイトでも「rootディレクトリ名まで書く」と言っているところが多いのに, それをあえて無視したらできてしまった.

ちょっと不安だけど一応解決！

追記（2017/1/9）

解決したと思ったら全然解決していませんでした…

確かにリンク先のアドレスは間違っていましたが, そもそもアドレスに日本語が含まれているのが問題で, その部分を対処すべきでした.

日本語を含むURLに関する対処方法はいろいろあるみたいですが,
面倒臭くなってしまったので記事自体のアドレスを変えることにしました.

若干アドレスが長くなり, 見ただけでは内容がわからなくなりましたが,
TwitterやFacebookに共有するときは, カードで内容が表示されるので問題ナッシング.

これで本当に解決！（多分）
